# 06_animation — 動畫展示階段

## 一、動畫階段在 pipeline 中的定位

本階段是分析 pipeline 的**最終展示層（presentation layer）**，位於靜態視覺化之後。
它不產生任何新的分析結果，也不改變資料，僅將前面階段（02_curated、03_processed、04_statistical）
已計算好的資料與統計量，以 **25fps 流暢動畫（GIF）** 的形式呈現，用於簡報、教學與資料敘事。

Pipeline 資料流向：

```
01_raw → 02_.. → 03_processed → 04_statistical → 05_static(靜態圖) → 06_animation(本階段)
```

- 每個動畫皆**一對一對應一張既有靜態圖**，只是把「一次看到全部」改成「隨時間逐步揭露」，
  讓觀者的注意力跟著資料的邏輯順序移動。
- 資料來源皆為上游階段的產物，動畫本身**唯讀**，不回寫任何資料。

## 二、動畫清單表

共 10 個動畫，皆為 25fps GIF，對應靜態圖、資料來源、動法與資料意義如下：

| # | 動畫 | 對應靜態圖 | 資料來源階段 | 動法 | 資料意義 |
|---|------|-----------|-------------|------|---------|
| anim01 | Lollipop 排名漸長 | 標的效應廣度排名圖 | 02_curated (curated_targets.csv，逐標的；summary_statistics.csv 僅供門檻/標註) | reveal（棒棒糖逐一漸長） | 各標的的效應廣度排名，由高至低逐一浮現 |
| anim02 | 條件補間 dot plot | 效應隨活化演變 dot plot | 03_processed (condition_stats) | tween（條件間平滑補間） | 效應量如何隨活化條件演變 |
| anim03 | QC 漏斗 count-up | QC 篩選漏斗圖 | 02_curated (curated_targets.csv 全體逐列做篩選計數；summary_statistics.csv 僅供最終計數標註) | count-up（數字滾動計數） | 品管篩選歷程 33,983 → 2,131 |
| anim04 | 泡泡漸入 | 效應 vs 廣度雙變數泡泡圖 | 02_curated (curated_targets.csv，逐標的) | reveal（泡泡逐一漸入） | 效應量與效應廣度的雙變數關係 |
| anim05 | ECDF 描繪 | 效應廣度累積分布圖 | 02_curated (curated_targets.csv，逐標的；summary_statistics.csv 僅供門檻/標註) | reveal（曲線逐段描繪） | 76% 的標的其 DE 基因數 < 10 |
| anim06 | 長條競賽 | 排名長條圖 | 03_processed (condition_stats) | tween（排名動態競賽） | 排名如何隨條件動態變化 |
| anim07 | 相關性熱圖格子漸入 | 相關性熱圖 | 02_curated (curated_targets.csv 的欄位相關) | reveal（格子逐格漸入） | n_DE 與 n_downstream 相關係數 1.00 |
| anim08 | 3D UMAP 旋轉 | 3D UMAP 散點圖 | 02_curated (curated_targets.csv，逐標的效應量級降維) | rotate（3D 視角旋轉） | 高效應標的在空間中聚集成團 |
| anim09 | 瀑布描繪 | 全標的效應瀑布圖 | 02_curated (curated_targets.csv，逐標的) | reveal（瀑布逐條描繪） | 全體標的的效應大小排序 |
| anim10 | 分歧長條 | 上/下調方向性分歧長條圖 | 02_curated (curated_targets.csv，逐標的 n_up_genes/n_down_genes) | reveal（左右分歧漸入） | 效應的方向性（上調 vs 下調） |

**動法定義：**
- **reveal（漸入揭露）**：元素依邏輯順序逐一或逐段出現。
- **tween（補間）**：在兩個資料狀態（如不同條件）之間做平滑內插過渡。
- **count-up（滾動計數）**：數值由起始滾動到終值，強調數量變化。
- **rotate（旋轉）**：3D 視角繞軸旋轉，用以觀察空間結構。

## 三、技術規格

| 項目 | 規格 |
|------|------|
| 影格率 | 25 fps（流暢，非跳格） |
| 緩動曲線 | ease-in-out（進出皆平滑加減速，避免機械等速） |
| 元素進場 | 漸入（fade / grow-in），避免突兀出現 |
| 格式 | GIF |
| 對應關係 | 每個動畫一對一對應一張靜態圖 |
| 資料性質 | 唯讀，不改動上游資料 |

## 四、重要提醒表（資料誠信註記）

以下兩個動畫在製作過程中對原始靜態圖做了**方法學上的修正**，務必留意，避免誤讀：

| 動畫 | 原始問題 | 修正做法 | 為何重要 |
|------|---------|---------|---------|
| **anim04 泡泡漸入** | 原本是**退化的火山圖**：以 \|效應\| 當作 y 軸，但資料**沒有 p-value 欄位**，缺少火山圖必要的統計顯著性維度，因此並非真正的火山圖。 | 已改為**泡泡圖**，改以「效應 vs 效應廣度」兩個實際存在的變數作圖。 | 火山圖的 y 軸必須是統計顯著性（如 −log10 p）。用 \|效應\| 冒充只會產生誤導性的「假火山圖」，改成泡泡圖如實呈現手上真正有的兩個變數。 |
| **anim08 3D UMAP 旋轉** | UMAP 是以**效應量級（effect magnitude）**做降維，而非表現譜。 | 保留此降維方式，但需正確解讀空間結構。 | 因為降維依據是效應量級，圖中**紅、藍兩色並非彼此分離**；真正的結構是「**高效應標的聚集在同一角落**」。不可按一般 UMAP 習慣解讀成「紅藍分群」。 |
