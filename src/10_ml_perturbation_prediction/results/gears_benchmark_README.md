# GEARS 基準測試 — 結果(誠實負面結果，且比 GenePT 基線更能說明問題）

**任務：** 用官方 `cell-gears` 套件（GO 共同註解相似度圖 + co-expression 相似度圖
+ GNN），對被擾動基因的 zero-shot（訓練時完全沒見過的基因）預測其對全部
18,129 個測量基因的表現量 profile。

**資料：** `GWCD4i.pseudobulk_merged.h5ad`，Rest 條件，`sc.pp.normalize_total`
+ `sc.pp.log1p` 標準化後，子抽樣至 2,000 個擾動基因（見
`gears_model/README.md` 的記憶體限制說明）+ 全部 3,387 個 ctrl 樣本。
`split="simulation"`，75% 基因訓練 / 25% 基因當 zero-shot 測試集（500 個
從未在訓練時被當成擾動看過的基因）。10 epochs，CPU，訓練用時 92.4 分鐘。

## 結果

| 指標 | GEARS | 均值基線 |
|---|---|---|
| Pearson（全部 18,129 個基因） | **0.9916** | **0.9956** |
| Pearson（僅顯著差異表現基因，官方 `pearson_de`） | **0.0210** | *(未計算——均值基線只跟全基因組比較)* |

**GEARS 贏了均值基線嗎？沒有。** 跟 `genept_baseline` 的結論一致：打不過
「訓練集平均 post-perturbation profile」這個簡單基線。

## 比數字更重要的發現：「全基因組 Pearson 0.99」是誤導性的高分

乍看 0.9916 像是很好的結果，但這個數字幾乎完全是「大多數基因根本沒被這個
擾動顯著影響」撐起來的——pseudobulk profile 裡，任何兩個條件之間，
**絕大多數基因的表現量根本沒什麼變化**，所以只要預測「一個大致合理的平均
表現量 profile」，不管有沒有抓到「這個特定擾動的特定效應」，全基因組
Pearson 就會很高（這正是為什麼**均值基線**——完全不看是哪個基因被擾動的
天真預測——也能拿到 0.9956，比 GEARS 還高）。

**真正衡量「這個模型有沒有學到擾動特異性訊號」的指標是官方
`pearson_de`（只看該擾動的顯著差異表現基因）：GEARS 只有 0.0210，
幾乎等於零、等於隨機。** 這比 `genept_baseline` 的結果更直接地說明問題：
不是「模型稍微輸給基線」，而是「在真正重要的基因子集上，模型幾乎沒有
任何預測力」。

## 這是 bug 嗎？不是——訓練/驗證 loss 過程完全正常收斂

見 `../results/gears_benchmark.json` 對應的完整訓練 log（10 epochs）：
train/validation Overall MSE 穩定在 0.055-0.062，Top 20 DE MSE 穩定在
0.246-0.261，沒有發散、沒有過擬合的異常跳動。模型確實「收斂」了——只是
收斂到的解跟均值基線在「全基因組尺度」上差不多，在「顯著差異基因」這個
真正有意義的子集上幾乎沒有訊號。這是誠實的負面結果，不是訓練失敗。

## 跟 2025-2026 文獻的關係

這個「全基因組相關性看起來很高、但顯著差異基因子集上模型幾乎沒有訊號」
的現象，正是 Ahlmann-Eltze et al.（*Nature Methods* 2025）等基準測試論文
反覆強調的方法學陷阱之一：只看全基因組 Pearson/MSE 容易產生虛假的樂觀
印象，必須額外報告「僅看顯著差異基因」的子集指標才能看到模型有沒有真的
學到擾動特異性的訊號。GEARS 官方套件本身就內建了 `pearson_de` 這個指標
（而非我們自己發明），這裡的誠實結果正好示範了為什麼這個子集指標
不可或缺。

## 這次執行的已知限制（誠實記錄，不是隱藏）

- **只用了 2,000 個擾動基因**（子抽樣自全量 11,287 個），因為
  `cell-gears` 官方實作在記憶體受限的機器上對全量資料的 pickle 步驟會讓
  可用記憶體從 30GB 打到 <1.1GB（見 `gears_model/README.md`）。
- **只跑了 10 epochs**，是「第一輪誠實驗證」而非充分調參後的最終結果——
  但給定 loss 已經穩定在同一個範圍震盪超過 5 個 epoch（見上），額外
  epoch 大概率不會實質改變「贏不了均值基線」這個結論。
- **只用 Rest 條件**，跟 `genept_baseline` 的評估範圍一致，未涵蓋
  Stim8hr/Stim48hr。
- 過程中修了五個 `cell-gears` 官方套件與新版 pandas/scipy/networkx 的
  相容性問題、一個真正的方法學問題（原始 pseudobulk count 未標準化導致
  訓練發散），全部記錄在 `../gears_model/README.md`——這些修正不影響
  結果的誠實性，只是讓程式能跑起來。

## 兩個獨立方法（GenePT embedding+Ridge、GEARS GO-graph+GNN）都誠實地
## 沒有打贏均值基線，這件事本身就是這次探索最重要的結論

`genept_baseline`（簡單方法）與 `gears_model`（複雜方法，官方實作）
用完全不同的架構做同一類任務，卻得到方向一致的負面結果——這比任何單一
模型的正面結果都更有參考價值，直接呼應 2025-2026 年整個領域的基準測試
共識：目前不論簡單或複雜的方法，用基因身分/嵌入去預測未見過擾動的下游
效應，都還沒有真正可靠的方法。這不代表「任務 A 萃取的全基因組帶符號
DE 矩陣沒用」——它讓這兩個誠實的基準測試得以存在，本身就是價值；只是
說明「用它訓練監督式 ML 模型去預測未見基因的效應」這條路，至少在這兩個
有代表性的方法上，還沒有找到突破口。
